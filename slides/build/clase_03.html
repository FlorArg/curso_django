Vistas

Basicamente una vista o view es un "tipo" de página web en nuestra aplicación, generalmente provee una función específica. Generalmente tiene una plantilla asociada.

En su forma mas simple una view es una funcion en Python que recibe el request como parametro y luego de realizar alguna tarea genera y retorna un objeto response.

Mapeo de URLs

En Django el contenido se entrega a partir de views, y el encargado de determinar qué view se ejecuta es un módulo Python informalmente llamado URLconfs.

Estos módulos definen como se mapean las URL (expresiones regulares) a funciones (las views).

1 mysite/
2     myapp/
3         ...
4         views.py
5         urls.py  <--- Agregamos

En url.py argeamos la tupla que mapea la url en nuestra view

1 from django.conf.urls import patterns, url
2 
3 from myapp import views
4 
5 urlpatterns = patterns('',
6     url(r'^(\w+)$', views.index, name='index')
7 )

Incluir URLs

Luego de crear un modulo al nivel de aplicacion el próximo paso es incluirlo en el URLconf raíz.

1 mysite/
2     myapp
3     mysite/
4         ...
5         settings.py
6         urls.py  <---- URLconf raíz

Importamos include

1 from django.conf.urls import patterns, include
2 
3 urlpatterns = patterns('',
4     url(r'^myapp/', include('myapp.urls')),
5     url(r'^admin/', include(admin.site.urls)),
6 )

Obteniendo información de la URL

Utilizando expresiones regulares se pueden asociar partes de la URL a parámetros de la URL.

1 from views import listado_historico
2 
3 urlpatterns = patterns('',
4     url(r'^listado_historico/(\d{4})-(\d{1,2})(\d{1,2})',
5     listado_historico, name='listado_historico'),
6 )

y la vista

1 from datetime import date
2 
3 def listado_historico(request, anio, mes, dia):
4     try:
5         fecha = date(int(fecha), int(mes), int(dia))

Usando plantillas

Creamos un template en templates/myapp/index.html

1 {% if modelos %}
2     <ul>
3     {% for modelo in modelos %}
4         <li>{{ modelo }}</li>
5     {% endfor %}
6     </ul>
7 {% else %}
8     <p>Sin datos.</p>
9 {% endif %}

Modificamos la vista index en myapp/views.py

1 from django.http import HttpResponse
2 from django.template import Context, loader
3 from myapp.models import Modelo
4 from django.shortcuts import render_to_response
5 
6 def index(request):
7     context = {'modelos': Modelo.objects.all()}
8     return render_to_response('myapp/index.html', context)

Herencia esqueleto

Una de las partes mas poderosas del motro de plantillas es la herencia.

El mecanismo consiste en definir una base o esqueleto de plantilla y mediante la herencia redefinir bloques o partes de ese esqueleto.

 1 <!DOCTYPE html>
 2 <html lang="es">
 3 <head>
 4     <link rel="stylesheet" href="style.css" />
 5     <title>{% block title %}Mi sitio{% endblock %}</title>
 6 </head>
 7 <body>
 8     <div id="sidebar">
 9         {% block sidebar %}
10         <ul>
11             <li><a href="/">Inicio</a></li>
12         </ul>
13         {% endblock %}
14     </div>
15     <div id="content">
16         {% block content %}{% endblock %}
17     </div>
18 </body>
19 </html>

Instalación y Uso

Se instala con pip en nuestro ambiente

1 pip install django-extensions

Se agrega como una aplicación instalada en setting.py en la lista de INSTALLED_APPS.

1 INSTALLED_APPS = ( 'django_extensions', )

Para ver los nuevos comandos disponibles

1 python manage.py help

Algunos comandos requieren la instalación de otras aplicaciones

  • export_emails will require the python vobject module to create vcard files.

  • graph_models requires pygraphviz to render directly to image file.
